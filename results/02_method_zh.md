# 方法論 (Methodology)

## 系統架構概述

我們提出的 SimLens 系統採用三層架構,旨在實現可追溯、可解釋的短視頻推薦模擬。整個系統的核心理念是將「影片客觀特徵」和「用戶行為權重」分開處理,通過數學化的方式建立兩者之間的明確關聯。與傳統的黑盒神經網絡方法不同,SimLens 保證 100% 的可追溯性,使每個預測結果都可以分解為具體特徵的貢獻。系統架構如圖 1 所示。

```
┌─────────────────────────────────────────────────────────────┐
│                    SimLens 系統架構                          │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  第一層: 客觀影片特徵提取 (Objective Feature Extraction)     │
│  ┌───────────────────────────────────────────────────┐      │
│  │ 輸入: 影片片段                                     │      │
│  │ 處理: 自動化特徵計算                              │      │

```

## 第一層: 客觀影片特徵提取

第一層負責從視頻中自動提取五個客觀、可計算的特徵。這些特徵的選擇基於視頻分析領域的研究文獻,每個特徵都可以通過開源工具完全自動化地計算,無需人工標註或主觀評估。

### 1.1 平均鏡頭時長 (Average Shot Duration, ASD)

**定義**: 視頻中所有鏡頭的平均持續時間(秒)。

**計算方法**: 使用 PySceneDetect 進行自動場景檢測。PySceneDetect 通過分析相鄰幀之間的內容差異來識別鏡頭邊界。

**數學公式**:
```
ASD = (T_total) / (N_shots)
```
其中:
- T_total: 視頻總時長(秒)
- N_shots: 檢測到的鏡頭總數

**標準化**: 將原始 ASD 值映射到 1-10 分制:
```
ASD_score = 10 - min(9, (ASD - 1) / 1.0)
```

**分數解釋**:
- 1-3分: 長鏡頭(ASD > 8秒),節奏緩慢
- 4-6分: 中等鏡頭長度(ASD 4-8秒),節奏適中
- 7-10分: 短鏡頭(ASD < 4秒),節奏快速

**研究依據**: 電影研究顯示,平均鏡頭時長從 1930 年代的 12 秒降至現代的 2-4 秒,反映了觀眾對視覺節奏的偏好變化。

### 1.2 鏡頭切換率 (Shot Change Rate, SCR)

**定義**: 每分鐘的鏡頭切換次數。

**計算方法**: 基於 ASD 直接計算。

**數學公式**:
```
SCR = 60 / ASD
```

**標準化**: 將 SCR 映射到 1-10 分制:
```
SCR_score = min(10, (SCR - 2) / 2.8)
```

**分數解釋**:
- 1-3分: 低切換率(< 6 cuts/min),視覺變化少
- 4-6分: 中等切換率(6-15 cuts/min),視覺變化適中
- 7-10分: 高切換率(> 15 cuts/min),視覺變化頻繁

**研究依據**: 高切換率與觀眾注意力維持相關,快節奏剪輯能提高短視頻的吸引力。

### 1.3 光流強度 (Optical Flow Magnitude, OFM)

**定義**: 視頻幀間的運動強度,反映畫面中物體或鏡頭的運動程度。

**計算方法**: 使用 OpenCV 的 Farneback 光流算法計算相鄰幀之間的光流向量。

**數學公式**:
```
OFM = (1/N) Σ sqrt(u²_i + v²_i)
```
其中:
- N: 像素總數
- u_i, v_i: 像素 i 的水平和垂直光流分量

**標準化**: 將 OFM 映射到 1-10 分制:
```
OFM_score = min(10, OFM / 10)
```

**分數解釋**:
- 1-3分: 靜態畫面(OFM < 30),幾乎無運動
- 4-6分: 中等運動(OFM 30-70),適度的物體或鏡頭運動
- 7-10分: 高強度運動(OFM > 70),快速運動或劇烈鏡頭變化

**研究依據**: 運動強度是視覺刺激的重要指標,影響觀眾的注意力和參與度。

### 1.4 語音密度 (Speech Density, SD)

**定義**: 視頻中包含語音的時間比例。

**計算方法**: 使用 Whisper 語音識別模型檢測語音片段。

**數學公式**:
```
SD = (T_speech / T_total) × 100%
```
其中:
- T_speech: 包含語音的總時長(秒)
- T_total: 視頻總時長(秒)

**標準化**: 將 SD 映射到 1-10 分制:
```
SD_score = SD × 10
```

**分數解釋**:
- 1-3分: 低語音密度(< 30%),主要是背景音樂或環境音
- 4-6分: 中等語音密度(30-60%),語音與其他音頻混合
- 7-10分: 高語音密度(> 60%),持續的語音內容

**研究依據**: 語音密度反映信息傳遞的強度,高語音密度通常意味著高信息量。

### 1.5 內容複雜度 (Content Complexity, CC)

**定義**: 視頻視覺內容的多樣性和豐富度。

**計算方法**: 使用 CLIP 模型提取視頻幀的視覺特徵向量,計算特徵向量的標準差作為複雜度指標。

**數學公式**:
```
CC = std(CLIP_features)
```
其中:
- CLIP_features: 從視頻幀提取的 CLIP 特徵向量集合
- std: 標準差函數

**標準化**: 將 CC 映射到 1-10 分制:
```
CC_score = min(10, CC × 20)
```

**分數解釋**:
- 1-3分: 低複雜度(CC < 0.2),視覺內容單一
- 4-6分: 中等複雜度(CC 0.2-0.4),視覺內容有一定變化
- 7-10分: 高複雜度(CC > 0.4),視覺內容豐富多樣

**研究依據**: 視覺多樣性反映內容的豐富程度,影響觀眾的視覺興趣。

### 1.6 工具與實現規格

所有特徵提取工具均為開源且公開可用,確保研究的可重現性和透明度。

**PySceneDetect**:

**Whisper**:
- 模型: base 或 small
- 語言: 自動檢測
- 任務: transcribe
- 開源許可: MIT

**CLIP**:
- 模型: ViT-B/32
- 採樣率: 每秒 1 幀
- 特徵維度: 512
- 開源許可: MIT

## 第二層: 用戶權重學習

第二層負責從真實用戶行為數據中學習用戶特定的特徵權重。與傳統的基於規則或專家知識的方法不同,我們直接從用戶的觀看歷史中學習其偏好模式。

### 2.1 數據來源: MicroLens-100K

**數據集統計**:
- 用戶數: 100,000
- 視頻數: 19,738
- 交互記錄: 719,405
- 關鍵信息: 每條交互記錄包含用戶 ID、視頻 ID、觀看時長

**數據預處理**:
1. 過濾掉觀看時長 < 1 秒的記錄(可能是誤觸)
2. 計算每條記錄的留存率: retention = watch_time / video_duration
3. 為每個視頻提取五個特徵: [ASD, SCR, OFM, SD, CC]

### 2.2 權重學習方法

**問題定義**: 對於用戶 u,學習權重向量 w_u = [w₁, w₂, w₃, w₄, w₅],使得預測的留存率與實際留存率最接近。

**數學模型**: 線性回歸
```
retention_predicted = w₁×ASD + w₂×SCR + w₃×OFM + w₄×SD + w₅×CC + b
```

**訓練過程**:
1. 將用戶的觀看歷史分為訓練集(80%)和驗證集(20%)
2. 使用最小二乘法求解權重:
```
w_u = argmin Σ(retention_actual - retention_predicted)²
```
3. 在驗證集上評估性能

**正則化**: 為防止過擬合,使用 L2 正則化:
```
w_u = argmin [Σ(retention_actual - retention_predicted)² + λ||w||²]
```
其中 λ = 0.01

**替代方法**: 除了線性回歸,也可以使用:
- Ridge Regression: 更強的 L2 正則化
- Lasso Regression: L1 正則化,可產生稀疏權重
- Gradient Boosting: 捕捉非線性關係

### 2.3 權重解釋

學習到的權重向量反映了用戶的內容偏好:

**正權重 (w_i > 0)**: 該特徵值越高,用戶越可能觀看更長時間
- 例如: w_SCR = 0.8 表示用戶偏好高切換率的快節奏視頻

**負權重 (w_i < 0)**: 該特徵值越高,用戶越可能提前離開
- 例如: w_ASD = -0.5 表示用戶不喜歡長鏡頭的慢節奏視頻

**零權重 (w_i ≈ 0)**: 該特徵對用戶的觀看決策影響不大
- 例如: w_SD = 0.1 表示用戶對語音密度不敏感

### 2.4 用戶特質量化與權重映射

為了支持創作者自定義代理人(例如「喜歡刺激、沒耐心的 20 歲大學生」),我們需要將用戶特質量化為可測量的指標,並建立從特質到影片特徵權重的映射關係。這個過程確保代理人的行為基於真實數據,而非 LLM 的幻覺。

#### 2.4.1 用戶特質量化框架

我們採用多維度的特質量化框架,整合心理學和推薦系統研究中的成熟模型:

**Big Five 人格特質** (Costa & McCrae, 1992):

1. **Openness (開放性)**: 對新體驗的開放程度 (0-100 分)
   - 高開放性用戶偏好多樣化、創新內容
   - 與影片特徵關聯: 高開放性 → 高 CC (內容複雜度) 權重

2. **Conscientiousness (盡責性)**: 組織性和目標導向程度 (0-100 分)
   - 高盡責性用戶偏好教育性、結構化內容
   - 與影片特徵關聯: 高盡責性 → 高 SD (語音密度) 權重

3. **Extraversion (外向性)**: 社交活躍度和能量來源 (0-100 分)
   - 高外向性用戶偏好社交、娛樂性內容
   - 與影片特徵關聯: 高外向性 → 高 OFM (光流強度) 和 SD 權重

4. **Agreeableness (親和性)**: 合作和同理心程度 (0-100 分)
   - 高親和性用戶偏好溫馨、正面情感內容
   - 與影片特徵關聯: 影響內容主題偏好

5. **Neuroticism (神經質)**: 情緒穩定性 (0-100 分)
   - 高神經質用戶可能避免高壓力、緊張內容
   - 與影片特徵關聯: 低神經質 → 高 OFM 權重

**社交特質** (基於 Agent4Rec, Zhang et al., 2024):

1. **Activity (活躍度)**: 
   ```
   T_activity(u) = Σ y_ui (用戶觀看的影片總數)
   ```
   - 分層: 低活躍 (< 33%), 中活躍 (33-66%), 高活躍 (> 66%)
   - 影響: 高活躍用戶觀看時間更長,容忍度更高

2. **Conformity (從眾性)**:
   ```
   T_conformity(u) = (1/N) Σ |r_ui - R_i|²
   ```
   - 分層: 低從眾 (獨特品味), 中從眾, 高從眾 (跟隨主流)
   - 影響: 低從眾用戶對小眾內容接受度更高

3. **Diversity (多樣性)**:
   ```
   T_diversity(u) = |∪ G_i| (用戶觀看的影片類型總數)
   ```
   - 分層: 低多樣性 (專注特定類型), 中多樣性, 高多樣性
   - 影響: 高多樣性用戶對不同風格內容接受度更高

**人口統計學特徵**:

- **年齡**: 18-24, 25-34, 35-44, 45-54, 55+
  - 研究依據: 年輕用戶偏好快節奏內容 (高 SCR 和 OFM 權重)
  - 中年用戶偏好信息密集內容 (高 SD 權重)

- **職業**: 學生、教育工作者、科技業、創意產業、服務業等
  - 影響: 職業背景影響內容主題偏好

- **興趣領域**: 科技、藝術、運動、音樂、旅遊、美食等
  - 影響: 直接影響內容主題偏好

#### 2.4.2 從特質到權重的映射方法

我們提出兩種數據驅動的映射方法,確保權重基於真實用戶行為:

**方法 1: 基於 MicroLens-100K 的聚類映射**

```python
# 步驟 1: 對 MicroLens-100K 用戶進行聚類
user_weights = [learn_weights(user) for user in microlens_users]
kmeans = KMeans(n_clusters=10)
clusters = kmeans.fit(user_weights)

# 步驟 2: 為每個聚類標註特質
cluster_traits = {
    0: {'age': '18-24', 'activity': 'high', 'big5_extraversion': 70},
    1: {'age': '25-34', 'activity': 'medium', 'big5_openness': 80},
    ...
}

# 步驟 3: 新用戶映射到最近的聚類
def map_persona_to_weights(persona_traits):
    nearest_cluster = find_nearest_cluster(persona_traits, cluster_traits)
    weights = cluster_centers[nearest_cluster]
    return weights
```

**方法 2: 基於 PersonaChat 的特質提取與映射**

PersonaChat 數據集 (Zhang et al., 2018) 包含 8,000+ 個 persona 描述,Big5-Chat (arXiv:2410.16491v1) 包含 100,000 個基於 Big Five 的對話。我們使用這些數據集來訓練特質提取和映射模型:

```python

def extract_persona_traits(persona_description):
    """
    輸入: "我是一個 25 歲的學生,喜歡動作電影..."
    輸出: {
        'age': 25,
        'occupation': 'student',
        'big5_openness': 75,
        'big5_extraversion': 60,
        ...
    }
    """
    traits = llm_extract_traits(persona_description)
    return traits

# 步驟 2: 訓練映射函數
def train_trait_to_weight_mapping(microlens_data):
    """
    從 MicroLens-100K 學習特質到權重的映射
    """
    X = []  # 特質向量
    y = []  # 權重向量
    
    for user in microlens_data:
        traits = extract_user_traits(user)  # 從觀看歷史推斷特質
        weights = learn_user_weights(user)  # 從觀看時長學習權重
        X.append(traits)
        y.append(weights)
    
    # 使用神經網絡或回歸模型
    mapping_model = train_regression_model(X, y)
    return mapping_model

# 步驟 3: 為新 persona 生成權重
def generate_weights_for_persona(persona_description):
    traits = extract_persona_traits(persona_description)
    weights = mapping_model.predict(traits)
    return weights
```

#### 2.4.3 冷啟動處理

對於新用戶(無觀看歷史),使用以下策略:

**方法 1: 全局平均權重**
```
w_new = (1/N) Σ w_u
```
使用所有用戶的平均權重作為初始值

**方法 2: 基於特質的聚類權重**
1. 提取新用戶的特質 (年齡、職業、Big Five 等)
2. 找到特質最相似的用戶聚類
3. 使用該聚類的平均權重

**方法 3: 基於 PersonaChat 的權重生成**
1. 使用 LLM 從用戶描述中提取特質
2. 通過訓練好的映射模型生成權重
3. 隨著用戶互動,逐步更新權重

這種方法的優勢在於:
- ✅ **有數據支持**: 權重來自真實用戶行為,不是 LLM 幻覺
- ✅ **可追溯**: 可以解釋為什麼某個特質對應某個權重
- ✅ **可擴展**: 支持任意用戶特質組合
- ✅ **可驗證**: 可以通過實驗評估映射準確性

## 第三層: 留存率計算

第三層將視頻特徵和用戶權重結合,計算最終的留存率預測分數。

### 3.1 留存率計算公式

對於用戶 u 觀看視頻 v,留存率分數計算如下:

```
Retention_Score = Σ(Feature_i × Weight_i) + b
                = w₁×ASD + w₂×SCR + w₃×OFM + w₄×SD + w₅×CC + b
```

其中:
- Feature_i: 視頻 v 在第 i 個特徵上的標準化分數(1-10)
- Weight_i: 用戶 u 在第 i 個特徵上的權重
- b: 偏置項(bias term)

### 3.2 分數歸一化

為了使分數更易解釋,我們將原始分數歸一化到 [0, 1] 區間:

```
Retention_Score_normalized = sigmoid(Retention_Score_raw)
                           = 1 / (1 + e^(-Retention_Score_raw))
```

**分數解釋**:
- 0.0-0.3: 低留存,用戶可能會快速跳過
- 0.3-0.7: 中等留存,用戶會觀看但可能不會看完
- 0.7-1.0: 高留存,用戶很可能完整觀看

### 3.3 時序分析

對於長視頻,我們可以將其分割為多個片段(例如每 10 秒一個片段),分別計算每個片段的留存率,生成留存率曲線:

```
Retention_Curve = [Score_t1, Score_t2, ..., Score_tn]
```

這使我們能夠:
1. 識別視頻中的高潮和低谷
2. 預測用戶可能離開的時間點
3. 為內容創作者提供優化建議

### 3.4 可追溯性優勢

SimLens 的核心優勢是 100% 的可追溯性。每個留存率預測都可以完全分解為各個特徵的貢獻:

**示例分解**:
```
視頻 A 對用戶 B 的留存率預測: 0.75 (高留存)

詳細計算:
- ASD 貢獻: 8 × 0.3 = 2.4
- SCR 貢獻: 9 × 0.8 = 7.2  ← 主要正面因素
- OFM 貢獻: 7 × 0.5 = 3.5
- SD 貢獻: 4 × (-0.2) = -0.8
- CC 貢獻: 6 × 0.4 = 2.4
- 偏置項: 1.0
總分: 15.7 → sigmoid(15.7) ≈ 0.75

解釋: 用戶 B 非常看重鏡頭切換率(權重 0.8),而視頻 A 的切換率很高(9分),
因此預測高留存。語音密度的負貢獻被其他特徵的正貢獻抵消。
```

**與黑盒方法的對比**:

| 方面 | SimLens | 深度神經網絡 |
|------|---------|-------------|
| 可解釋性 | 完全可追溯 | 黑盒 |
| 特徵貢獻 | 明確量化 | 無法分解 |
| 調試能力 | 可識別問題特徵 | 難以調試 |
| 用戶信任 | 高(可解釋) | 低(不透明) |
| 計算成本 | 低 | 高 |

### 3.5 實際應用場景

**場景 1: 內容創作者優化**
創作者上傳視頻後,系統分析:
- "您的視頻在第 15-25 秒留存率較低(0.3),主要因為鏡頭切換率過低(3分)"
- "建議: 在該片段增加場景切換或添加動態元素"

**場景 2: 個性化推薦**
推薦系統為用戶推薦視頻時:

- 優先推薦高留存率(> 0.7)的視頻
- 避免推薦低留存率(< 0.3)的視頻

**場景 3: 用戶行為分析**
平台分析用戶群體:
- "年輕用戶(18-25歲)平均 SCR 權重為 0.9,偏好快節奏內容"
- "中年用戶(35-50歲)平均 SD 權重為 0.7,偏好信息密集內容"

## 系統整體流程

### 4.1 離線階段

**步驟 1: 視頻特徵提取**
- 對視頻庫中的所有視頻提取五個特徵
- 存儲特徵向量到數據庫
- 時間複雜度: O(N_videos)

**步驟 2: 用戶權重學習**
- 對每個用戶從其觀看歷史學習權重
- 存儲權重向量到數據庫
- 時間複雜度: O(N_users × N_history)

### 4.2 在線階段

**步驟 1: 檢索候選視頻**
- 使用協同過濾或內容過濾獲取候選集
- 候選集大小: 100-1000 個視頻

**步驟 2: 留存率預測**
- 對每個候選視頻計算留存率分數
- 時間複雜度: O(N_candidates) - 非常快速

**步驟 3: 排序與推薦**
- 按留存率分數降序排序
- 返回 Top-K 視頻(K = 10-50)

### 4.3 性能優化

**特徵緩存**: 預計算並緩存所有視頻的特徵向量

**權重更新**: 定期(每天或每週)更新用戶權重,而非實時更新

**批量計算**: 使用矩陣運算批量計算多個視頻的留存率:
```
Scores = Features_matrix × Weights_vector
```

**分佈式計算**: 特徵提取可以並行化處理,支持大規模視頻庫

## 總結

SimLens 的三層架構實現了以下目標:

1. **客觀性**: 第一層使用完全自動化的特徵提取,無需主觀評估
2. **數據驅動**: 第二層從真實用戶行為數據學習權重,而非依賴假設
3. **可追溯性**: 第三層保證每個預測都可以完全分解和解釋
4. **可擴展性**: 整個系統可以高效處理大規模視頻和用戶數據
5. **實用性**: 為內容創作者和推薦系統提供可操作的洞察

這種設計使 SimLens 成為一個既科學嚴謹又實用可行的視頻推薦模擬系統。
