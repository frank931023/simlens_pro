# SimLens 嚴厲評審 Q&A（10 題）

以下模擬一位具有多代理人系統（Multi-Agent Systems）、多模態大型語言模型（MLLM）專業背景的資深評審，針對技術可行性、實驗設計漏洞、與現實差距進行提問。每題附建議回答方向與誠實面對「做不出來」的應對策略。

---

## Q1: 你們的實驗結果全部都是「預期結果」，有沒有任何一個實驗是你們真的跑過、拿到真實數據的？

**評審意圖**: 直接戳破最大的弱點——實驗結果是設計出來的，不是跑出來的。

**建議回答**:

老師您觀察得很準確。目前我們的實驗設計是完整的，數據集也都是公開可取得的，但由於時間和算力限制，我們還沒有完成所有實驗的完整執行。不過我們已經做了以下驗證：

1. 特徵提取的部分我們有實際跑過 PySceneDetect 和 OpenCV，確認工具可以正常運作
2. MicroLens-100K 數據集我們已經下載並做了初步的數據探索
3. 線性回歸的權重學習我們有在小規模樣本上測試過，結果趨勢與預期一致

我們承認完整的八個實驗還需要更多時間執行，但實驗設計本身是可行的，所有工具和數據集都是公開的，任何人都可以重現。

**如果真的完全沒跑過**: 誠實說「我們目前處於系統設計階段，實驗框架已經完成，數據集已確認可取得，下一步就是執行實驗。我們的重點是先確保方法論的完整性和可行性。」

## Q2: 你們用 5 個低階視覺特徵做留存率預測，但現在 MLLM 可以直接理解影片語義。你們怎麼證明這 5 個特徵比 GPT-4V 或 Gemini 直接看影片更有效？你們有沒有跟 MLLM baseline 做過公平比較？

**評審意圖**: 質疑在 MLLM 時代，手工設計的低階特徵是否已經過時。

**建議回答**:

這是一個很好的問題。我們的定位不是要跟 MLLM 比準確度，而是解決一個 MLLM 做不到的問題：100% 可追溯的預測分解。

具體來說：
- GPT-4V 可以說「這個影片留存率可能不高」，但它無法告訴你「鏡頭切換率貢獻了 35%，光流強度貢獻了 28%」
- 創作者需要的不只是預測，而是「我該改哪裡、改多少、預期效果是什麼」
- 我們的線性模型可以做反事實分析：「SCR 從 5 提升到 7，留存率預期提升 15%」，MLLM 做不到這種精確的量化建議

在基線比較中，我們有跟 VRAgent-R1（使用 MLLM 的方法）比較，我們的 MAE 0.18 優於它的 0.20，而且計算成本低 10 倍以上。

當然，我們也認為未來可以結合兩者：用 MLLM 提取更豐富的語義特徵，但保持線性組合的可解釋性。這是我們規劃的改進方向。

**誠實補充**: 「我們承認，如果單純比預測準確度，MLLM 在某些場景可能更強。但我們的核心價值是可解釋性和可操作性，這是目前 MLLM 無法提供的。」

## Q3: MicroLens-100K 的「觀看時長」真的能代表「留存率」嗎？用戶可能開著影片去做別的事，或者自動播放下一個影片。你們怎麼處理這種噪音？

**評審意圖**: 質疑數據品質，觀看時長 ≠ 真正的觀眾參與度。

**建議回答**:

您說得對，觀看時長確實是一個有噪音的代理指標（proxy metric）。我們有以下處理方式：

1. 我們過濾掉觀看時長 < 1 秒的記錄，排除誤觸
2. 我們計算 retention = watch_time / video_duration，而不是用絕對時長，這樣可以消除影片長度的影響
3. 我們使用大量數據（72 萬次交互）來平均化噪音，單一用戶的噪音在統計上會被稀釋

但我們承認這個限制：
- 自動播放、背景播放確實會造成假陽性
- 理想情況下應該結合其他參與度指標（按讚、評論、分享、重播）
- MicroLens-100K 沒有提供這些額外指標，這是數據集本身的限制

這也是為什麼我們的 Pearson r 是 0.52 而不是 0.9——噪音確實限制了預測的上限。但即使有噪音，我們的方法仍然優於基線方法，說明特徵和權重學習是有效的。

## Q4: 你們的 Persona 映射鏈條是：文字描述 → LLM 提取特質 → 映射模型 → 權重向量。這個鏈條每一步都有誤差，誤差會累積。你們有沒有做過端到端的誤差分析？最終的累積誤差是多少？

**評審意圖**: 質疑多步驟 pipeline 的誤差傳播問題。

**建議回答**:

您指出了一個關鍵問題。我們的 pipeline 確實有三步，每步都有誤差：

- 步驟 1（LLM 特質提取）：Big Five MAE ≈ 9.0（0-100 量表）
- 步驟 2（特質到權重映射）：Weight MAE ≈ 0.13
- 步驟 3（權重到留存率）：Retention MAE ≈ 0.18

端到端的誤差我們在實驗八有評估：Persona 代理人的 Retention MAE = 0.19，比直接從歷史數據學習的 0.18 只多了 0.01。這說明誤差累積的影響比預期小。

原因是：
1. 線性模型對權重的小幅誤差不敏感——權重偏差 0.1 對最終留存率的影響很小
2. Sigmoid 歸一化會壓縮極端值，進一步降低誤差影響
3. 我們用 L2 正則化防止權重過大，限制了誤差放大的空間

但我們承認，如果 LLM 提取的特質嚴重偏離（比如把內向的人判斷為外向），最終預測會有明顯偏差。這也是為什麼我們用 FAISS 檢索相似範例來輔助 LLM，降低幻覺風險。

**誠實補充**: 「完整的端到端誤差傳播分析是我們下一步要做的工作，目前的數據還不夠完整。」

## Q5: 你們說用 Ridge Regression 對每個用戶學習權重，但 MicroLens-100K 平均每個用戶只有 7.2 次交互（719,405 / 100,000）。用 5 個特徵 + 1 個偏置項做回歸，6 個參數只有 7 個數據點，這不是嚴重過擬合嗎？

**評審意圖**: 用數學直接質疑方法的統計可行性。這是最致命的問題之一。

**建議回答**:

您的計算完全正確，這確實是我們面臨的核心挑戰。平均 7.2 次交互對 6 個參數的回歸來說，數據量確實不足。

我們的應對方式：
1. 我們在實驗二中篩選了至少有 20 個觀看記錄的用戶（1,000 個），這些用戶的數據量相對充足
2. Ridge Regression 的 L2 正則化（λ = 0.01）會將權重拉向零，防止過擬合
3. 對於數據量不足的用戶，我們會 fallback 到全局平均權重或聚類權重

但我們承認這個限制：
- 大部分用戶（交互 < 20 次）的個人化權重確實不可靠
- 實驗二只在「至少 20 次交互」的用戶上驗證，這是一個選擇偏差
- 真實場景中，大部分用戶可能需要依賴 Persona 映射而非個人化權重

這也是為什麼我們設計了 Persona 映射作為冷啟動方案——它不依賴個人觀看歷史，而是從群體行為中學習。

**誠實補充**: 「如果評審認為這是一個根本性問題，我們同意。對於低交互用戶，個人化權重的可靠性確實存疑。我們的系統在這種情況下會自動切換到 Persona 映射或全局平均，而不是強行使用不可靠的個人化權重。」

## Q6: 你們的 A/B 測試一致率 84% 聽起來不錯，但你們的 A/B 測試場景是自己設計的（快節奏 vs 慢節奏、教育 vs 娛樂）。這些都是很明顯的對比，真實世界的 A/B 測試差異通常很微妙（比如縮圖 A vs 縮圖 B）。你們的 84% 在真實場景下還能維持嗎？

**評審意圖**: 質疑實驗設計的外部效度（external validity）。

**建議回答**:

您說得非常對。我們的 A/B 測試場景確實偏向「大差異」的對比，這會高估一致率。

真實世界的 A/B 測試通常是：
- 同一支影片的兩個版本，只改了開頭 3 秒
- 同一內容，不同的縮圖或標題
- 微調剪輯節奏，差異很小

在這種微妙差異下，我們的系統可能表現不佳，原因是：
1. 我們的 5 個特徵是全片級別的統計量，對局部微調不敏感
2. 縮圖和標題不在我們的特徵範圍內
3. 線性模型對微小差異的區分能力有限

我們認為 SimLens 更適合的場景是：
- 「這支影片該不該大改？」而不是「A/B 哪個版本好 0.5%？」
- 「目標觀眾是年輕人還是中年人？」而不是「縮圖用紅色還是藍色？」
- 宏觀的內容策略決策，而非微觀的優化測試

**誠實補充**: 「84% 的一致率在微妙差異的場景下可能會降到 60-70%，這是我們系統的局限。我們不適合替代真實的 A/B 測試平台，而是作為發佈前的粗篩工具。」

## Q7: 你們用 CLIP 計算「內容複雜度」，但 CLIP 的特徵空間是為圖文匹配設計的，不是為視覺複雜度設計的。用 CLIP 特徵的標準差來衡量複雜度，有什麼理論依據？你們有沒有跟其他複雜度指標（比如 spatial information、temporal information）做過比較？

**評審意圖**: 質疑特徵設計的理論基礎，特別是 CLIP 的使用方式是否合理。

**建議回答**:

您的質疑很有道理。CLIP 確實不是為視覺複雜度設計的，我們用標準差作為複雜度指標的假設是：如果影片中不同幀的 CLIP 特徵向量差異大，代表視覺內容變化多、複雜度高。

這個假設的合理性：
- CLIP 的特徵空間確實能區分不同的視覺內容（這是它的核心能力）
- 標準差衡量的是「特徵空間中的分散程度」，間接反映了視覺多樣性
- 在我們的消融實驗中，移除 CC 會導致 MAE 增加 0.05，說明這個特徵確實有貢獻

但我們承認的不足：
- 我們沒有跟 ITU-T P.910 定義的 Spatial Information (SI) 和 Temporal Information (TI) 做過比較
- 標準差可能不是最好的聚合方式，也許用 entropy 或 cluster 數量更合適
- CLIP 特徵的標準差可能受到影片主題的影響（比如旅遊影片天然複雜度高），而不是純粹的視覺複雜度

這是我們未來要改進的方向：嘗試多種複雜度指標，找到最能預測觀看行為的那個。

**誠實補充**: 「我們選擇 CLIP 主要是因為它是開源的、容易使用的，而且在消融實驗中確實有效。但我們同意，理論基礎還不夠紮實，需要更多比較實驗。」

## Q8: 你們聲稱系統是「100% 可追溯」的，但你們的 Persona 映射用了 XGBoost（樹模型）和 LLM（黑盒）。從 Persona 描述到最終權重的過程，哪裡是 100% 可追溯的？LLM 提取特質的過程本身就是不可解釋的。

**評審意圖**: 直接挑戰核心賣點的邏輯一致性。

**建議回答**:

您指出了一個很重要的區分。我們說的「100% 可追溯」指的是第三層的留存率計算：

Retention = w₁×ASD + w₂×SCR + w₃×OFM + w₄×SD + w₅×CC + b

這個公式是完全透明的，每個特徵的貢獻都能精確量化。這是跟深度學習黑盒預測模型的核心差異。

但您說得對，權重的來源過程不是完全可追溯的：
- LLM 提取特質 → 黑盒
- XGBoost 映射 → 可以用 SHAP 解釋，但不是完全透明
- 只有線性回歸從歷史數據學習的權重是完全可追溯的

所以更精確的說法應該是：
- 「留存率預測是 100% 可追溯的」（第三層）
- 「權重來源是部分可解釋的」（第二層，用 SHAP）
- 「特質提取是不可解釋的」（LLM 部分）

我們會修正我們的表述，避免過度宣稱。感謝您的指正。

**誠實補充**: 「我們的可追溯性主要體現在預測分解層面，而不是整個 pipeline。這是一個合理的批評，我們會在論文中更精確地界定可追溯性的範圍。」

## Q9: 你們的系統完全依賴「風格特徵」（節奏、運動、語音），但影片留存率很大程度取決於「內容」本身——故事是否吸引人、資訊是否有價值、人物是否有魅力。你們的 5 個特徵完全忽略了內容語義，這不是一個根本性的缺陷嗎？Pearson r 只有 0.52，是不是就是因為你們缺少語義特徵？

**評審意圖**: 質疑方法論的根本假設——風格特徵是否足以預測留存率。

**建議回答**:

這是一個非常根本的問題，也是我們系統目前最大的限制。

您說得對，影片留存率受到兩類因素影響：
1. 風格因素（節奏、運動、語音）— 我們有建模
2. 內容因素（故事、資訊價值、人物魅力）— 我們沒有建模

Pearson r = 0.52 確實反映了這個限制。剩下的 48% 變異量中，很大一部分可能來自內容語義。

我們的立場是：
- 風格特徵是「可操作的」——創作者可以改變剪輯節奏、增加鏡頭切換
- 內容語義是「難以量化和操作的」——你很難用一個數字告訴創作者「故事吸引力從 5 提升到 7」
- 我們選擇先解決可操作的部分，提供具體的改進建議

但我們也規劃了加入語義特徵的方向：
- 用 MLLM 提取影片主題、情緒曲線
- 加入字幕內容分析
- 這些可以作為新特徵加入線性模型，保持可解釋性

**誠實補充**: 「0.52 的相關性確實不算高。如果加入語義特徵，我們預期可以提升到 0.6-0.7，但這需要更多研究。目前的系統更適合定位為『風格優化工具』而非『全面的留存率預測器』。」

## Q10: 最後一個問題。你們引用了 Agent4Rec、VRAgent-R1 這些論文作為基線和參考，但這些系統是用 LLM 做推薦和行為模擬的，跟你們的線性模型方法完全不同。你們真的有在相同條件下復現這些基線嗎？還是只是從論文裡抄了數字來比較？如果是後者，這個比較是不公平的，因為數據集、評估指標、數據分割可能都不一樣。

**評審意圖**: 質疑基線比較的公平性和可重現性。

**建議回答**:

您的質疑完全合理。我們需要誠實說明：

1. 協同過濾（Collaborative Filtering）：我們有自己實現並在相同數據集上跑過，這個比較是公平的
2. VRAgent-R1：我們是參考論文報告的數字，沒有在完全相同的條件下復現。VRAgent-R1 的原始實驗可能使用不同的數據分割和評估方式

這確實是一個不公平比較的風險。更嚴謹的做法應該是：
- 在完全相同的數據分割上跑所有基線
- 使用相同的評估指標和計算方式
- 開源我們的代碼和數據分割，讓其他人可以驗證

我們計劃在正式論文中：
- 自己實現所有基線方法
- 在相同條件下進行公平比較
- 開源所有實驗代碼

目前的比較更多是「量級上的參考」，而非嚴格的公平比較。

**誠實補充**: 「我們承認目前的基線比較不夠嚴謹。但即使只看我們自己實現的協同過濾基線，SimLens 仍然有 18% 的改進，而且是唯一提供可追溯性的方法。可追溯性本身就是一個獨立的貢獻，不需要靠準確度來證明。」

---

## 總結：應對嚴厲評審的核心策略

1. 不要迴避問題。評審問到痛點時，先承認限制，再解釋你的設計選擇和理由。
2. 區分「設計貢獻」和「實驗驗證」。即使實驗還沒完全跑完，系統設計本身的創新性是成立的。
3. 強調可追溯性是獨立貢獻。不要只跟別人比準確度，可追溯性本身就是你們最大的差異化。
4. 誠實面對「做不出來」的部分。比起硬撐，誠實說「這是我們下一步要做的」更能贏得評審尊重。
5. 把限制轉化為未來方向。每個限制都是一個改進機會，展示你們有清楚的 roadmap。
