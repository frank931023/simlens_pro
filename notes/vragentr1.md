VRAgent-R1 

abstract

現有研究多用 prompte-based 的模擬方式 並用凍結 LLM 的
方式，因面對多模態內容理解複雜挑戰，導致用戶學習效果不
好，最終限制了推薦性能

提出了 VRAgent-R1 在使用者模擬中加入人類
智慧的基於代理人架構，主要有兩個代理人

IP 代理人基於多模態大語言模型（MLLMs）模擬類人
的漸進式思考，有效地捕捉影片中隱藏的推薦語義

US 代理人基於深度的思維鏈（CoT）推理來精煉推
薦的影片集，並透過強化學習實現與真實用戶偏好的
更好對齊。

method

Item Perception Agent (IP Agent)

Key Frame Retrieval

因為輸入整支影片會帶來多於資訊導致低效效率，因此他會選擇最有用的視
覺表達來確保算法的效率。所以他改從影片均勻採樣10貞然後給 CLIP 計
算採樣貞和影片標題之間的視覺文本相似度分數，最後保留前三高分數。

Collaborative Multimodal Perception (CMP)

在提取到檢索的貞後，下一步識別影片中所傳達的具體事件和高層次語義。
由於某些標題並不直接反映影片的主題，我們將檢索到的幀與標題同時輸入
到 MLLM 中，並提示其理解標題所隱含的語義語境。在此過程中，MLLM 可
以提供標題的相關解釋並提供補充信息。原本單一模態的嵌入無法獨立捕捉
到如此細微的語義，現在清晰地理解影片信息和影片中的情感。

Recommendation Relevant Analysis

MLLM 一開始生成的文字不一定完美適合，畢竟可能有多餘解釋或幻
覺。所以他們 prompt the model 去分析影片細節和場景特色，然後模
型將影片內容重新改寫（reformulates）為簡潔且精準的描述。

User Simulation Agent
(US Agent)

對於一個有 N 個行為的使用者，前面 N-1 個行為被用來建立 user profile，第 N 個被
當作預測目標。根據使用者歷史偏好用 SASRec 推薦十支影片，模擬一個粗略的 recall
過程。因為處理多個影片與文本序列對 MLLM 來說有有挑戰性，IP 代理人會先多模態
影片轉換為文本格式，讓後續  US 代理人處理長文本格式。對於 RFT 過程，他們是先
透過 prompt 方式給予特別狀態(偏好和情緒) 並讓他們透過思索練  COT reasoning 方
式分析候選影片並執行相應的操作。

SASRec 是一個粗略的影片推薦模型，但它在這個實驗
中扮演的是「初步篩選」的角色。這 10 個影片裡，有
的可能真的很合胃口，有的可能普通。

task and reward

用戶偏好判斷

代理人會從候選列表中獲得一個項目，隨即被
問是否喜歡這隻推薦影片，會要求模型按照格
式回答，若回答符合格式則加一分。此外，使
用後處理函數解析標籤並轉為合法動作，檢查
是否與 GT 一致。正確的模擬將 Rjud = 1, 錯
誤則 -1

下一支影片選擇

代理人先審視所有候選項目，並透過提示
工程選擇用戶最可能接著觀看影片。因為
任務更複雜，正確選擇正樣本影片分配 2
分。

Group Relative Policy Optimization (GRPO)
群體相對策略優化

原本的強化學習中，需要兩個模型同時運作:
- actor: 負責生成回答的模型
- critic: 負責給演員打分的模型

GRPO: Ai = (ri - mean(r) ) / std(r)
系統不再問"這個好不好?"，而是問"這個回答在一群人中，是不是高於
平均水準?"

直接比較一組候選回答，不需要 critic model 來評估性能。
給予模型一個問題，他會進行採樣以生成一組不同回答。每個回答涉
及對用戶不同的 COT 推理及最終答案，我們計算對應獎勵
